# Быстрый старт: Защита ветки main

Этот файл содержит краткие инструкции по настройке защиты ветки `main` от прямых изменений.

## Автоматическая настройка (1 минута)

```bash
# 1. Создайте токен: https://github.com/settings/tokens
#    Права: repo (полный доступ)

# 2. Установите токен
export GITHUB_TOKEN=your_token_here

# 3. Запустите скрипт
./scripts/apply-branch-protection.sh
```

## Что будет настроено

После выполнения скрипта:
- ✅ Требуется pull request для всех изменений
- ✅ Требуется одобрение от владельца кода
- ✅ Требуется прохождение CI проверок
- ✅ Запрещены прямые коммиты в main
- ✅ Запрещён force push
- ✅ Правила применяются к администраторам

## Процесс работы после настройки

1. Создайте новую ветку:
   ```bash
   git checkout -b feature/my-feature
   ```

2. Внесите изменения и закоммитьте:
   ```bash
   git add .
   git commit -m "Add new feature"
   ```

3. Отправьте ветку:
   ```bash
   git push origin feature/my-feature
   ```

4. Создайте Pull Request на GitHub

5. Дождитесь прохождения проверок и одобрения

6. Выполните слияние через GitHub
