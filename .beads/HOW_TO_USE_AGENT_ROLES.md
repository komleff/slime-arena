# Промпты для активации ролей AI-агентов

Шпаргалка по активации ролей из [AGENT_ROLES.md](./AGENT_ROLES.md).

---

## Формат промпта

```
Ты [Роль]. Прочитай AGENT_ROLES.md секция "[номер и название]".
Задача: [описание]
```

---

## Роли

| Роль | Секция | Когда использовать |
|------|--------|-------------------|
| **Project Manager** | 0️⃣ PM | Координация спринта, делегирование, ревью-оркестрация, завершение сессии |
| **Architect** | 1️⃣ Architect | Декомпозиция эпиков, создание задач в Beads |
| **Developer** | 2️⃣ Developer | Написание кода, исправление багов |
| **Reviewer** | 3️⃣ Reviewer | Проверка PR, валидация качества |

> **Примечание:** Полная документация PM — в [PM_ROLE.md](./PM_ROLE.md). Art Director — в [ART_DIRECTOR_ROLE.md](./ART_DIRECTOR_ROLE.md).

---

## Примеры промптов

**PM:**
```
Ты PM. Прочитай AGENT_ROLES.md секция "0️⃣ Project Manager".
Задача: запустить ревью PR #65, собрать консенсус от Opus/Gemini.
```

**Architect:**
```
Ты Architect. Прочитай AGENT_ROLES.md секция "1️⃣ Architect".
Задача: декомпозировать Stage D Testing на подзадачи.
```

**Developer:**
```
Ты Developer. Прочитай AGENT_ROLES.md секция "2️⃣ Developer".
План утвержден. Задача: реализуй slime-arena-pgf (лимиты рекламы).
```

**Reviewer:**
```
Ты Reviewer. Прочитай AGENT_ROLES.md секция "3️⃣ Reviewer".
Задача: проверь PR #60 на соответствие требованиям.
```

---

## Вердикты ревью

| Вердикт | Действие |
|---------|----------|
| `APPROVED` | Можно мержить (нужно 3+ от разных ревьюверов) |
| `CHANGES_REQUESTED` | Ждать исправлений |
| `ESCALATION` | Сменить модель или эскалировать человеку |
