## Обзор

## Новые возможности

### Спрайтовая система (21 спрайт)

Полный редизайн визуальной идентификации игроков. Вместо 4 цветных кругов (`slime_green/blue/red/yellow`) — 21 уникальный спрайт с привязкой к профилю.

**Что изменилось:**

- **shared/src/sprites.ts** — единый источник: `SPRITE_NAMES` (21 спрайт), `pickSpriteByName()` (хеш по имени), `isValidSprite()` (whitelist-валидация)
- **Player.spriteId** — новое поле в Colyseus schema (`@type("string")`), синхронизируется с клиентом
- **JoinTokenPayload.spriteId** — спрайт передаётся через join-token при входе в матч
- **Matchmaking** — spriteId проходит через весь flow: `POST /matchmaking/join` → `MatchmakingService` → `JoinTokenService` → `ArenaRoom`
- **Leaderboard** — `isValidSprite()` валидация, fallback на `slime-base.webp`
- **Клиент** — рендеринг по `player.spriteId`, fallback на `pickSpriteByName(name)` для ботов

**Удалено:**

- `config/skins.json` — ошибочный артефакт (4 цветных скина)
- `generateRandomBasicSkin()` из `server/src/utils/` — перемещён в `meta/utils/`
- Дублированные `hashString()`/`pickSpriteForPlayer()` в `client/src/main.ts`

## Исправления ошибок

### P0 — Блокирующие

- **OAuth intent на MainScreen** (`a2c7f91`)
  Кнопка OAuth на главном экране использовала `intent="convert_guest"`, который требует `claimToken` (доступен только на экране результатов). Гости не могли войти с главного экрана. Исправлено: `intent="login"`.

### P1 — Критические

- **Зависание экрана выбора класса** (`ceb5b6e`, slime-arena-b7z6)
  При нажатии «Играть ещё» экран выбора класса зависал — таймер не обновлялся. Причина: гонка между `room.leave()` и `connectToServer()`. Исправлено: сброс `activeRoom = null` до `setPhase("connecting")`.

- **Скин не сохраняется при OAuth upgrade** (`ba1af70`, slime-arena-vsn5)
  При привязке OAuth skinId не передавался в claimToken → скин сбрасывался. Исправлено: проброс skinId через `matchResults → claimToken → auth/upgrade`.

- **404 при OAuth для новых пользователей** (`885392d`, slime-arena-ias0)
  При первом OAuth-входе (без гостевого аккаунта) сервер возвращал 404 «User not found» вместо создания аккаунта. Исправлено: автосоздание аккаунта в OAuth resolve.

### P2 — Важные

- **Таймер зависает на Chrome mobile** (`efe9960`, slime-arena-hfww)
  `setInterval` замерзал при уходе вкладки в background. Исправлено: абсолютное время через `Date.now()` + обработчик `visibilitychange`.

- **Фаза 'connecting' мелькает главным экраном** (`b2869e6`, slime-arena-3v3o)
  При «Играть ещё» на долю секунды показывался главный экран. Исправлено: `ConnectingScreen` со спиннером.

- **normalizeNickname() падает на null/undefined** (`6075177`, slime-arena-n17m)
  `normalizeNickname(null)` → строка `'null'` проходила валидацию. Исправлено: guard `if (nickname == null) return ''`.

- **Модификаторы укуса несимметричны** (`1ecd828`, slime-arena-mtw)
  `damageBonusMult` применялся только к `attackerGain`, `damageTakenMult` только к `scatterMass`. Исправлено: единый `combinedMult` для обоих.

- **Талант Вампир не по GDD** (`e55dbe7`, slime-arena-4xh)
  Формула не соответствовала GDD («бок 10%→20%, хвост 15%→25%»). Исправлено: бонус = `defenderMass × bonusPct × combinedMult`.

- **Гость видит PLAYER после матча** (`69de6d9`, slime-arena-y2z2)
  `useState(authService.isAnonymous())` не пересчитывался. Исправлено: `useComputed()` из сигнала `currentUser`.

## Технический долг

- **generateRandomBasicSkin()** (`56c002e`) — `Math.random()` перемещён из `server/src/utils/` в `server/src/meta/utils/` (формально вне симуляции, но физически нарушал правило детерминизма)
- **Дедупликация** (`396425c`) — удалены локальные `hashString()`/`pickSpriteForPlayer()` из `main.ts`, заменены на `pickSpriteByName` из shared
- **Import fix** (`a48add7`) — убрано расширение `.js` из import (CommonJS модуль)

## Безопасность

- **Валидация skinId** — whitelist через `getBasicSkins()` для гостей, `isValidSprite()` для leaderboard

## Известные ограничения

- **slime-arena-vk4m** (P1): Спрайтовая система не имеет сквозного flow «выбрал → сохранил → всегда видит». Клиент не передаёт skinId в matchmaking, `clearGuestData()` удаляет `guest_skin_id`, нет API для смены скина. Отложено в отдельную задачу.
- **slime-arena-ef98** (P3): формула Вампира — неоднозначность в GDD «20% от массы жертвы».
