## Исправления ошибок

### P1 — Критические

- **Зависание экрана выбора класса** (slime-arena-b7z6)
  При нажатии «Играть ещё» экран выбора класса зависал — таймер не обновлялся. Причина: гонка между `room.leave()` и `connectToServer()`. Исправлено: сброс `activeRoom = null` до `setPhase("connecting")`, таймаут безопасности 10 сек.

- **Скин не сохраняется при OAuth upgrade** (slime-arena-vsn5)
  При привязке OAuth-аккаунта к гостю skinId не передавался в claimToken → скин сбрасывался на дефолтный. Исправлено: проброс skinId через цепочку `matchResults → claimToken → auth/upgrade`.

### P2 — Важные

- **Таймер зависает на Chrome mobile** (slime-arena-hfww)
  `setInterval` с декрементом замерзал при уходе вкладки в background. Исправлено: абсолютное время через `Date.now()` + обработчик `visibilitychange` для пересчёта при возврате.

- **Фаза 'connecting' мелькает главным экраном** (slime-arena-3v3o)
  При «Играть ещё» на долю секунды показывался главный экран. Исправлено: новый компонент `ConnectingScreen` со спиннером «Подключение...».

- **normalizeNickname() падает на null/undefined** (slime-arena-n17m)
  `normalizeNickname(null)` превращал null в строку `'null'`, которая проходила валидацию. Исправлено: guard `if (nickname == null) return ''`.

- **Модификаторы укуса применяются несимметрично** (slime-arena-mtw)
  `damageBonusMult` применялся только к `attackerGain`, `damageTakenMult` — только к `scatterMass`. Таланты «Стойкий» и «Острые зубы» работали неполноценно. Исправлено: общий `combinedMult` применяется симметрично к обоим слагаемым.

- **Талант Вампир не по GDD** (slime-arena-4xh)
  Вампир перенаправлял scatter в attackerGain, но формула не соответствовала GDD («бок 10%→20%, хвост 15%→25%»). Исправлено: бонус = `defenderMass * bonusPct * combinedMult`, ограничен доступным scatter.

- **Гость видит PLAYER после матча** (slime-arena-y2z2)
  `useState(authService.isAnonymous())` не пересчитывался после возврата из матча → гость видел «PLAYER» и медали вместо «Войти». Исправлено: реактивное вычисление через `useComputed()` из сигнала `currentUser`.

## Технический долг

- **Изолировать generateRandomBasicSkin()** (slime-arena-vpti)
  `Math.random()` в `server/src/utils/generators/skinGenerator.ts` нарушал правило детерминизма (формально в мета-слое, но физически в utils). Перемещён в `server/src/meta/utils/skinGenerator.ts`.

## Безопасность

- **Валидация skinId для гостей**
  Добавлена проверка skinId через whitelist `getBasicSkins()` — гости не могут подставить ID премиум-скина в claimToken.

## Известные ограничения

- `slime-arena-ef98` (P3): формула Вампира использует `attacker.mass` для базового gain и `defender.mass` для бонуса. Требуется уточнение GDD — «20% от массы жертвы» или «20% = base(от своей) + bonus(от жертвы)». Текущая реализация совпадает с предыдущим поведением.
