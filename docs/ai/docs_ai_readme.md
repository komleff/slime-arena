# docs/ai — как использовать промпты в репозитории

Эта папка хранит **промпты для ИИ‑кодера** и рабочие планы. Они лежат в GitHub, чтобы:
- версионироваться вместе с кодом;
- быть доступными из VS Code;
- обеспечивать единый стиль постановки задач.

## Куда положить
Папка: `docs/ai/`

Рекомендуемые файлы:
- `docs/ai/SlimeArena_Flight_AI_Coder_Prompt.md` — общий «контейнер‑промпт» с правилами.
- `docs/ai/SlimeArena_Flight_MVP_Server_Task.md` — промпт на реализацию **серверного MVP** (Assist → Physics → Collision + тесты).
- `docs/ai/SlimeArena_Flight_Client_Smoothing_Task.md` — промпт на **клиентское сглаживание**.
- `docs/ai/SlimeArena_Flight_Implementation_Plan.md` — рабочий план (может устареть после завершения).

Рядом рекомендуется хранить неизменяемое ТЗ:
- `docs/SlimeArena_Flight_TZ_Merged.md` (экспорт из канвы)

## Как использовать в VS Code
1) Открой нужный файл из `docs/ai/`.
2) Скопируй содержимое целиком.
3) Вставь в чат ИИ‑кодера:
   - сначала **контейнер‑промпт**
   - затем **задачу (MVP Server / Client Smoothing)**
   - затем приложи/вставь текст ТЗ (или дай файл `docs/SlimeArena_Flight_TZ_Merged.md`).

## Правило
Промпты — рабочий инструмент. Их можно менять и улучшать.
ТЗ — «вечный» документ: менять только при изменении требований.
