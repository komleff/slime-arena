Роль: ИИ-кодер. Цель: реализовать серверную и клиентскую часть управления/физики Slime Arena строго по ТЗ “SlimeArena Flight TZ (Merged)” (канва проекта).

ЖЁСТКИЕ ПРАВИЛА
1) Язык реализации: как в проекте (TypeScript/Node для сервера, TS для клиента), без сторонних физдвижков.
2) Серверная симуляция: фиксированный dt = 1/30, 30 Гц. Запрещён переменный deltaTime.
3) Порядок систем на серверном тике неизменен: FlightAssistSystem → PhysicsSystem → CollisionSystem.
4) Физика ньютоновская. В симуляции использовать ТОЛЬКО силы/моменты. Ускорения — только справочные для редактора.
5) Режимы FA:OFF/FA:ON, клавиатура, второй джойстик, BRAKE, тороидальный мир, выбор цели — НЕ реализовывать.
6) Углы/угловые скорости внутри симуляции — в радианах. Любые параметры с суффиксами Deg/Degps конвертировать при загрузке.
7) Масса может меняться в полёте. При мгновенном изменении массы сохранять vel и angVel (аркадное правило из ТЗ).
8) Коллизии: импульсный метод + позиционная коррекция + итерационный решатель. Использовать restitution из WorldPhysicsConfig. Скорости не клэмпить.
9) FlightAssist обязан реализовать: (а) поворот по yawCmd (с yawCmdEps), (б) drift-brake за comfortableBrakingTimeS, (в) мягкое гашение перескорости overspeedDampingRate, (г) демпфирование угловой скорости за angularStopTimeS с ограничением моментом.
10) Turn torque scaling: по умолчанию exp=1.5 (обоснование через I ∝ m²). Не менять без явного указания.
11) Deadzone: применить на клиенте ДО отправки и повторно на сервере.
12) Ввод: inputSeq монотонный, сервер применяет latestInput; если нет ввода > inputTimeoutMs, считать mag=0.

ФОРМАТ ВЫДАЧИ РЕЗУЛЬТАТА
- Сначала перечисли модули/файлы, которые создашь/изменишь.
- Затем реализуй модули по очереди. Код должен быть самодостаточный, компилируемый, с типами.
- Для каждого модуля: кратко (5–10 строк) объясни ответственность и вход/выход (без воды).
- Обязательно добавь минимальные автотесты/проверки, покрывающие Acceptance Criteria из раздела 14 (хотя бы smoke-tests).
- Любые спорные места решай строго по ТЗ. Если параметр не задан — используй дефолт из ТЗ.

ПЕРВЫЙ ИНКРЕМЕНТ (MVP-веха)
Реализовать серверную часть: конфиги + FlightAssist + Physics + Collision + серверный тик-пайплайн + минимальные тесты.
Клиентское сглаживание можно как второй инкремент, но зафиксируй интерфейсы снапшота, чтобы оно подключилось без переделок.

Никаких дополнительных фич, оптимизаций или “улучшений” сверх ТЗ.
