## 1. КРИТИЧНО ВАЖНЫЕ ПРАВИЛА

### 1.1. Безопасные зоны (Safe-Area) [ИЗ UI-TZ §2.3]
Все элементы управления должны учитывать:
- **Верхний отступ:** минимум 24px (под системную панель/«чёлку»)
- **Нижний отступ:** минимум 34px (под навигацию/жесты)
- **Боковые отступы:** минимум 16px от краёв экрана

**Влияет на:** кнопки «Назад/Закрыть», панели HUD, нижнюю навигацию.

### 1.2. Форматы и размеры [ОБЪЕДИНЕНО]

| Тип asset           | Формат                                 | Размеры                                  | Плотность    | Особые требования                                |
| ------------------- | -------------------------------------- | ---------------------------------------- | ------------ | ------------------------------------------------ |
| **Иконки UI**       | SVG (предпочтительно) или PNG с альфой | 24×24, 32×32, 48×48                      | Вектор/2x/3x | Без текста, читаемые на малых размерах           |
| **Кнопки**          | PNG с альфой                           | 200×60 (основные), 150×50, круглые 64×64 | 1x, 2x       | Обязательно состояния: Normal, Pressed, Disabled |
| **Панели/карточки** | PNG/WebP с 9-slice                     | Разные (см. таблицы)                     | 2x           | Скругления 8px/16px/24px, мягкие тени            |
| **Фоны экранов**    | PNG/JPG                                | 1920×1080                                | 1x, 2x, 3x   | Статичные, без важных элементов по краям         |
| **Эффекты**         | APNG/PNG последовательность            | По ситуации                              | 2x           | Прозрачность, цикличные анимации                 |

## 10. СПЕЦИАЛЬНЫЕ ИНСТРУКЦИИ ДЛЯ ИИ-ХУДОЖНИКОВ

### 10.1. Ключевые запреты

1. **Никакого текста в иконках** (кроме логотипов)
2. **Избегать фотореализма** — стиль мультяшный, яркий
3. **Не использовать чистый чёрный** (#000000) — заменять на тёмно-серый (#121212)
4. **Не использовать чистый белый** (#FFFFFF) для фонов — заменять на светлые оттенки (#F5F5F5)
5. **Избегать слишком мелких деталей** — элементы должны читаться на маленьких экранах

### 10.2. Рекомендации по генерации

1. **Промпты должны включать:**
   - Стиль: «flat design, cartoon style, vibrant colors»
   - Контекст: «mobile game UI icon for...»
   - Форма: «rounded corners, soft shadows»
   - Цвет: «green (#4CAF50) for Hunter class»

2. **Используемые инструменты:**
   - **MidJourney:** Для концепт-артов, фонов, сложных композиций
   - **Recraft:** Для иконок, векторных элементов
   - **Nano Banana/Sora1:** Для эффектов, анимаций, стилистической согласованности

3. **Проверка качества:**
   - Убедиться в прозрачности фона (alpha channel)
   - Проверить читаемость на тёмном и светлом фоне
   - Убедиться, что 9-slice элементы корректно масштабируются
