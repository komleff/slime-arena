## 2. Компоненты системы

**MetaServer** — мета-сервер. Отвечает за:
- Авторизацию (`auth`)
- Профиль игрока
- Кошелёк и магазин
- Матчмейкинг

**MatchServer** — игровой сервер. Отвечает за:
- Физическую симуляцию матча (Colyseus `ArenaRoom`)
- Результаты матча (`MatchResultService`)
- Валидацию `joinToken`

**Client** — браузерный клиент (Preact + Canvas). Отвечает за:
- Отображение арены
- Управление (джойстик)
- Подключение к `MatchServer` через WebSocket

---

## 3. Ключевые архитектурные принципы

1. Клиент отправляет только `InputCommand` — позиции и скорости не передаются.
2. `MatchServer` авторитативен — вся физика симулируется на сервере.
3. Любая операция с ценностью идемпотентна через `operationId`.
4. `MetaServer` — единственный источник правды для профиля и экономики.
5. Конфигурация передаётся через `RuntimeConfig` с версионированием (`configVersion`).

## 5. Правила изменения контрактов

- Любое изменение схемы БД → обновить `Part4.md`.
- Любое изменение HTTP API → обновить `Part4.md`.
- Любое изменение конфигурационной схемы → обновить `Part4.md` и `schemaVersion`.
- Нарушение этих правил = P0-дефект.
